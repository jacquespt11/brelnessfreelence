Spécifications Complètes - Plateforme SaaS Brelness
🎯 Objectif Général
Développer une application web SaaS multilocataire permettant la gestion de réservations de produits pour différents types de boutiques (vêtements, cosmétiques, électronique, etc.). La plateforme doit fonctionner en trois niveaux : Super Admin, Admin Boutique, et Client Visiteur.

📋 Modèles de Données
lisp

User (Super Admin / Admin Boutique)

├── Email

├── Password (hashed)

├── Role (superadmin/admin)

├── Profile (nom, photo, téléphone)

└── Preferences (thème, langue)



Shop

├── Name & Slug

├── Category

├── Owner (Super Admin)

├── Admin (Admin Boutique)

├── Logo & Banner (URL Cloudinary)

├── License (status, dates)

├── Contact Info

└── Social Media Links



Product

├── Name & Slug

├── Description & Images

├── Price & Stock

├── Shop (référence)

├── ShareLink (unique)

├── Reviews (référence)

└── Reservations Count



Reservation

├── Product (référence)

├── Customer (Name, Phone, Email)

├── Quantity

├── Status (pending/confirmed/completed)

├── CreatedAt & UpdatedAt

└── Notes



Review

├── Product (référence)

├── Rating (1-5)

├── Comment

├── CustomerName (optionnel)

└── IsModerated



Notification

├── User (référence)

├── Type (reservation/license/system)

├── Title & Message

├── Read Status

└── CreatedAt
